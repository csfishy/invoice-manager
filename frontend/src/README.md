# Frontend Source

Planned Next.js admin application.

Recommended future structure:

- `app/` for routes and layouts
- `components/` for reusable form, table, badge, and dialog components
- `features/` for bill management, licensing, auth, and dashboard screens
- `lib/` for API clients and shared utilities

UI priorities:

- admin-oriented forms and filters
- bill list and detail pages
- license status screen
- audit log and reporting views
